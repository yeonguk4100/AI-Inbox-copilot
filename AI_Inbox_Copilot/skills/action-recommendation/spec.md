# spec.md — Action Recommendation

## 목표

> 이메일 분석 결과를 종합하여 사용자가 지금 수행해야 하는 행동을 우선순위와 함께 추천한다.

---

## 맥락

### 대상

* 대학생
* 직장인
* 이메일을 자주 사용하는 일반 사용자

### 사용 목적

매일 수신된 메일들을 분석한 결과를 바탕으로, 사용자가 메일함을 열지 않고도 **각 메일에 대해 1분 이내에 실행 가능한 행동을 결정**할 수 있도록 돕는다.

---

## 범위

### 입력

* 메일 기본정보: ID, 제목, 발신자, 수신 시각
* Mail Summary 결과 (핵심 요약)
* Mail Classification 결과 (분류 카테고리)
* Priority Analysis 결과 (중요도: High/Medium/Low)
* Security Check 결과 (보안 위험도, 판단 근거)

### 출력

메일별 행동 추천 (Markdown 테이블 + 상세 설명)

포함 내용

* 메일별 정보 (제목, 발신자, 분류, 중요도, 보안 위험도)
* **추천 행동 (읽기 / 보관 / 삭제 / 주의 필요 / 즉시 조치)**
* **판단 근거 (중요도·보안·분류 기준을 조합하여 설명)**
* 사용자 확인 필요 여부 (Yes/No)

---

## 제약

* 실제 메일을 삭제하지 않는다.
* 실제 답장을 전송하지 않는다.
* 실제 캘린더에 등록하지 않는다.
* 실제 메일을 이동하지 않는다.
* **모든 행동은 추천만 제공**한다. (Human Checkpoint 유지)
* **중요도 High + Medium의 메일만 최대 5개까지만 선별**하여 행동 추천한다.
* **모든 행동 추천에 판단 근거를 필수 명시**한다. (확신 없는 판정 금지)
* 보안 위협 메일은 삭제를 자동 실행하지 않고, "**주의 필요**" 또는 "**차단 권고**"로만 표시한다.
* Markdown 테이블 + 상세설명 형식을 유지한다.
* 다음 Skill(morning-mail-brief)이 사용할 수 있도록 **메일별 구조화된 데이터**로 전달한다.
* 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.

## 출력 형식

### 파일명

ActionRecommendation.md

### 구조

#### 1. 메일별 행동 추천 요약 (Markdown 테이블)

중요도 및 보안 상태에 따라 최대 5개 메일을 선별하여 표 형식으로 제시

| 메일 | 발신자 | 중요도 | 보안 | 추천 행동 | 판단 근거 | 확인 필요 |
|------|--------|--------|------|----------|---------|----------|
| [제목](#id) | sender@domain | High | 안전 | 읽기 | 학교 공지사항 + 긴급도 높음 | No |
| … | … | … | … | … | … | … |

#### 2. 메일별 상세 행동 추천 (메일당 3~5줄)

```
**[메일 제목]**
- 발신자: sender@domain
- 분류: [학교/업무/금융/광고/보안/뉴스레터/기타]
- 중요도: [High/Medium/Low]
- 보안: [안전 / 주의 필요 / 위협 + 사유]
- **추천 행동:** [읽기 / 보관 / 삭제 / 주의 필요 / 즉시 조치]
- **판단 근거:** (중요도 + 보안 + 분류 기준을 조합하여 설명)
- 사용자 확인 필요: [Yes/No]
```

### 분량

* 테이블: 최대 5개 메일
* 메일별 상세설명: 3~5줄 (원문 숫자·일정은 정확히 인용)
* 전체: 1~2페이지 이내

---

## 성공 기준

* [ ] **중요도 High + Medium만** 선별하여 **최대 5개까지만** 행동 추천한다.
* [ ] 각 메일에 대해 **읽기 / 보관 / 삭제 / 주의 필요 / 즉시 조치 중 정확히 하나**를 선택하여 추천한다.
* [ ] **모든 추천에 판단 근거를 명시**한다. (중요도/보안/분류 기준 기반)
* [ ] 보안 위협 메일은 자동 삭제 대신 "**주의 필요**" 또는 "**차단 권고**"로만 표시한다. (Human Checkpoint)
* [ ] 사용자 확인 필요 여부를 Yes/No로 명확히 표시한다.
* [ ] 원문의 숫자(날짜, 금액, 시간)를 정확히 인용한다. (환각 금지)
* [ ] Markdown 테이블 + 상세설명으로 구성되어 Morning Mail Brief에서 즉시 사용 가능하다.
* [ ] 메일당 3~5줄 이내로 간결하게 작성한다.

---

## 추천 행동 정의

* **읽기:** 중요한 내용이 있어 반드시 확인 필요 (High 메일, 긴급 사항)
* **보관:** 참고용으로 보관 필요 (Medium 메일, 향후 참조 가능성)
* **삭제:** 확인할 필요 없음 (일반 광고, 중복 발송 등)
* **주의 필요:** 피싱 의심, 악성 링크 등 보안 위험 (사용자 판단 필요)
* **즉시 조치:** 계정 탈취 위험, 의료 응급 등 긴급 조치 필요 (최고 우선순위)
