# morning-mail-brief Skill Specification

## 목표

> 각 Skill의 분석 결과를 **5개 탭 구조의 브리핑 문서**로 통합하여, 사용자가 **1분 이내에 아침 메일의 핵심을 파악**하고 각 메일에 대한 행동을 결정할 수 있도록 제시한다.

---

## 맥락

### 목적

매일 아침 7시 자동 실행으로:
1. **지난 24시간 수신 메일의 종합 분석 결과를 1~2페이지 문서로 제공**
2. **긴급 처리 필요 메일을 상단에 강조**
3. **보안 경고를 별도 탭으로 분리하여 주목**
4. **영수증/결제 정보를 체계적으로 정리**
5. **사용자가 각 메일에 대해 즉시 행동 결정 가능하도록**

### 사용 흐름

1. 모든 Skill의 결과 수집:
   - security-check (보안 판정)
   - mail-summary (요약)
   - mail-classification (분류)
   - priority-analysis (중요도)
   - action-recommendation (권고 행동)
   - payment-analysis (결제 집계)
2. 5개 탭별로 메일과 정보를 정렬 및 포맷팅
3. Markdown 또는 HTML 문서로 최종 산출
4. 사용자에게 전달 (메신저 알림 + 자동화 패널)

---

## 범위

### 입력

모든 선행 Skill의 결과 통합:

* **security-check:** security_status, risk_types, verdict
* **mail-summary:** summary, keywords, key_dates, key_amounts
* **mail-classification:** main_category, sub_category
* **priority-analysis:** priority (High/Medium/Low), priority_score
* **action-recommendation:** recommended_action, action_reason, confirmation_needed
* **payment-analysis:** 누적 금액, 정기 결제, 이상 거래
* **메타 정보:** 수집 메일 수, 분석 기간, 절단 여부 (50건 초과)

### 출력

**최종 브리핑 문서** (Markdown 또는 HTML)

포함 내용:
* 5개 탭 구조 (아래 "5개 탭 정의" 참고)
* 메일별 3~5줄 요약 + 행동 추천
* 원문 숫자 정확도 100% 보장
* 판단 근거 명시

---

## 5개 탭 정의

### 탭 1: [수신 메일] — 개요

**목적:** 하루 메일 현황을 한눈에 파악

**포함 내용:**
* 24시간 내 수신한 **총 메일 건수**
* **분류별 건수** (학교 / 업무 / 금융 / 광고 / 뉴스레터 / 보안 / 기타)
* **중요도별 분포** (High / Medium / Low 각 몇 건)
* **처리 상태:** 
  - "N건 분석 완료" 
  - "50건 초과 시: 최신순 50건 분석, 총 OOO건 중 50건 처리"
* **분석 기간:** YYYY-MM-DD HH:MM ~ YYYY-MM-DD HH:MM

**예시:**
```
## [수신 메일]

- **총 건수:** 32건
- **분류 현황:**
  - 학교: 5건 | 업무: 8건 | 금융: 3건 | 광고: 7건 | 뉴스레터: 6건 | 보안: 2건 | 기타: 1건
- **중요도:**
  - High: 4건 | Medium: 11건 | Low: 17건
- **처리 상태:** 32건 모두 분석 완료 (2025-07-02 00:00 ~ 2025-07-03 00:00)
```

---

### 탭 2: [긴급/중요] — Action Required

**목적:** 즉시 또는 오늘 중 처리해야 할 메일만 강조

**선별 기준:**
* **중요도 High 메일** (최대 5개, 시간 순)
* 또는 **중요도 Medium 중 기한이 임박한 메일** (24시간 내)
* **보안 경고** 중 "주의 필요" / "차단 권고" 메일 (있으면 포함)

**포함 정보 (메일당):**
* ✓ 제목 (링크 + 메일 ID)
* ✓ 발신자 (메일 주소 + 이름)
* ✓ 분류
* ✓ 중요도 (High / Medium)
* ✓ 핵심 요약 (3~5줄)
* ✓ **추천 행동** (읽기 / 보관 / 삭제 / 주의 필요 / 즉시 조치)
* ✓ 판단 근거 (짧게)
* ✓ 마감 기한 (있으면)

**형식:** Markdown 테이블 또는 계층형 리스트

**예시:**
```
## [긴급/중요]

| # | 메일 | 발신자 | 분류 | 중요도 | 행동 | 기한 |
|----|------|--------|------|--------|------|------|
| 1 | [수강신청 안내] | korea@korea.ac.kr | 학교 | High | 읽기 | 3.12 |
| 2 | [프로젝트 리뷰 회의] | team@company.com | 업무 | High | 읽기 | 오늘 |

### 1. [수강신청 안내]
- **발신자:** 교무처 (korea@korea.ac.kr)
- **분류:** 학교 > 학사공지
- **핵심:** 2025년 봄학기 수강신청이 3월 10일~12일에 진행됩니다. 신청은 학사시스템에서 가능하며 선착순입니다.
- **행동:** 읽기 (일정 미리 확인, 신청 준비)
- **근거:** 학사 필수 일정, 24시간 내 마감
```

### 탭 3: [관심사/업무] — For Your Interest

**목적:** Medium 우선도의 참고 메일과 사용자 관심사 메일

**선별 기준:**
* **중요도 Medium 메일** (기한 임박하지 않은 것)
* **사용자 관심사 부합 메일** (금융 종목, 관심 뉴스레터 등)
* **해당 탭이 비어도 됨** (그 날은 생략)

**포함 정보:**
* 메일 제목 (링크 + 메일 ID)
* 발신자
* 분류 + 서브 분류
* 중요도
* 요약 (1~3줄)
* 관심사 연결 여부 (사용자 설정과 부합 여부)

**형식:** Markdown 리스트 또는 테이블

**예시:**
```
## [관심사/업무]

### 📈 금융 & 투자
- **[배당금 입금 알림]** (삼성증권)
  - Samsung Electronics 배당금 10,000원 입금됨
  - 📌 관심 종목: SSNLF

### 📰 뉴스레터
- **[주간 AI 트렌드]** (AI Weekly)
  - 이번 주 주요: GPT-5 릴리스, AI 규제 가이드라인
```

### 탭 4: [보안 경고] — Security Alerts

**목적:** 사용자 주의가 필요한 모든 보안 메일을 한곳에 정리

**선별 기준:**
* security-check 결과 **"주의 필요"** 또는 **"위험"** 판정 메일
* **모든 보안 카테고리 메일** (로그인 알림, 비밀번호 재설정, MFA 등)

**포함 정보 (메일당):**
* ✓ 제목 (메일 ID)
* ✓ 발신자 (도메인 검증 상태 표시)
* ✓ 보안 상태 (안전 / 주의 필요 / 위험)
* ✓ 위험 유형 (피싱 의심 / 사칭 / 악성 링크 / 의심 첨부 / 로그인 알림)
* ✓ 판단 근거 (도메인 검증, 본문 패턴, 링크, 첨부 분석)
* ✓ **권고 조치** (차단 권고 / 주의 후 읽기 / 삭제 권고)

**경고 수준 표시:**
* 🔴 **위험:** 차단 권고, 삭제 권고
* 🟡 **주의 필요:** 주의 후 읽기, 발신자 재확인
* 🟢 **안전:** 로그인 알림 등 정상 메일

**형식:** Markdown 목록 + 색상/이모지 강조

**예시:**
```
## [보안 경고]

### 🔴 위험

**[긴급] 계정 확인 필요 - PayPal**
- 발신자: paypa1.com (⚠️ 유사 도메인)
- 위험 유형: 피싱 의심
- 근거: 
  - 도메인 검증 실패 (PayPal ≠ paypa1.com)
  - 본문: "비밀번호 재설정 필요", "즉시 조치" 등 긴급성 조성
  - 링크: bit.ly 단축 URL, 목표 도메인 불명확
- **권고:** 🚫 차단 권고, 삭제 권고

### 🟡 주의 필요

**[알림] 새로운 기기에서 로그인**
- 발신자: accounts.google.com
- 위험 유형: 로그인 알림 (정상)
- 근거: 공식 Google 도메인, 표준 보안 알림 형식
- **권고:** 📖 주의 후 읽기 (자신의 로그인 확인 필요)
```

### 탭 5: [영수증] — 30일 누적 결제

**목적:** 한 달간의 결제/구독 내역을 한눈에 파악

**포함 정보:**
* 기간: YYYY-MM-DD ~ YYYY-MM-DD (최근 30일)
* **총 결제 금액 (누적)**
* **카테고리별 분류** (구독료 / 쇼핑 / 정기결제 / 기타)
* **정기 결제 항목** (월간 구독료 예상액)
* **이상 거래** (고액 또는 신규 결제)

**포함 정보 (결제 항목당):**
* 거래 날짜
* 항목명
* 금액
* 결제 수단 (신용카드 / 계좌이체 / 구독 등)
* 상태 (완료 / 예정)

**형식:** Markdown 테이블 + 요약

**예시:**
```
## [영수증] — 최근 30일 (2025-06-03 ~ 2025-07-02)

### 📊 누적 결제 요약

**총 결제 금액: 235,000원** (12건)

| 카테고리 | 금액 | 건수 |
|---------|------|------|
| 📱 구독료 | 45,000원 | 3 |
| 🛍️ 쇼핑 | 120,000원 | 5 |
| 📅 정기결제 | 50,000원 | 2 |
| 기타 | 20,000원 | 2 |

### 📌 정기 결제 항목 (월간 예상)

| 항목 | 금액 | 상태 |
|------|------|------|
| Netflix | 14,900원 | 활성 |
| Spotify | 10,900원 | 활성 |
| iCloud 저장소 | 2,900원 | 활성 |

**월간 예상 고정 결제: 38,700원**

### ⚠️ 주의 필요 거래

- **Apple One (신규 구독)** - 12,900원 (신규 항목)
- **Amazon Prime** - 8,900원 (예상 범위 초과)
```

## 제약

* **숫자 정확도 100%:** 모든 금액·날짜·시간은 원문과 정확히 일치
* **개인정보 마스킹:** 신용카드, 계좌, 주민번호 등 마스킹 처리
* **형식 일관성:** 모든 날짜는 YYYY-MM-DD, 금액은 OOO,OOO원 형식
* **한눈에 파악 가능:** 전체 문서 1~2페이지 (A4 기준)
* **메일별 3~5줄 제한:** 요약은 장황하지 않게 간결하게
* **링크 생성:** 클릭 가능한 메일 ID 또는 Gmail 링크 제공 (선택)
* **수신 메일 개수 명시:** 50건 초과 시 "OOO건 중 최신순 50건 분석" 표시

## 성공 기준

* [ ] **5개 탭이 모두 포함**되고 순서대로 배열되어 있다.
* [ ] **[수신 메일] 탭이 현황을 정확히 반영**한다 (총 건수, 분류별 분포, 중요도 분포).
* [ ] **[긴급/중요] 탭에 High + 기한 임박 Medium 메일만** 최대 5개 수록.
* [ ] **모든 메일의 숫자(날짜, 금액, 시간, 수량)가 원문과 100% 일치**.
* [ ] **[보안 경고] 탭에 모든 위험/주의 메일이 포함**되고, 판단 근거 명시.
* [ ] **[영수증] 탭의 누적 금액 계산이 정확**하고, 정기 결제가 식별됨.
* [ ] **전체 문서가 1~2페이지 이내** (A4 기준, 여백 포함).
* [ ] **각 메일별 설명이 3~5줄** 이내로 간결.
* [ ] **개인정보 마스킹 처리** (카드번호, 계좌, 주민번호 등).
* [ ] **한눈에 중요한 정보가 파악 가능** (스캔 가능한 구조, 강조, 이모지/색상 활용).
* [ ] **테스트 20건 메일 대상:** 누락 0건, 오류 0건.

## 레이아웃 팁

* **제목은 큰 글씨 + 이모지** (## 또는 ###)
* **테이블 사용:** 정렬된 정보는 테이블로
* **컬러/이모지 활용:** 중요도 시각화
  - 🔴 위험 / 🟡 주의 / 🟢 안전
  - 📌 중요 / 📰 정보 / 📅 일정
* **공백 활용:** 탭별로 단락 구분
* **링크:** 메일 ID 또는 Gmail 링크 생성 (선택사항)

---

## 배포 방법

* **자동화 패널:** 생성 후 패널에 기록
* **메신저 알림:** Telegram/Slack/KakaoTalk 연결 시 자동 알림 (문서 링크)
* **이메일:** (선택) 사용자 메일로 발송
* **스토리지:** Markdown 파일 저장 (워크스페이스 또는 클라우드)
