# spec.md — Priority Analysis
## 목표
> Mail Summary와 Mail Classification 결과를 기반으로, 사용자 관심사(전공 학사 일정, 관심 금융 티커 등)를 반영하여 이메일의 중요도를 High / Medium / Low로 평가하고 사용자가 먼저 확인해야 하는 메일인지 판단한다.
---
## 맥락
### 대상
* 대학생
* 직장인
* 이메일을 자주 사용하는 일반 사용자
### 사용 목적
중요한 메일을 빠르게 식별하여
사용자가 우선적으로 확인해야 하는 메일을 추천한다.
---
## 범위
### 입력
* Mail Summary 결과
* Mail Classification 결과
* 메일 제목
* 메일 본문
* 발신자
* 수신 시간
### 출력
Markdown 형식의 중요도 분석 결과
포함 내용
* 메일 정보
* 중요도(High / Medium / Low)
* 긴급도
* 중요도 판단 이유
* 우선 확인 여부
---
## 제약
* 중요도는 반드시 High, Medium, Low 중 하나로 표시한다.
* 중요도 판단 이유를 반드시 작성한다.
* 메일을 삭제하거나 이동하지 않는다.
* 중요도는 긴급성, 마감 여부, 학업·업무 관련성, 금융·투자 관련성, 사용자 관심사를 종합하여 판단한다.
* Markdown 형식을 유지한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.
---
## 중요도 판정 기준
중요도는 아래 신호를 종합하여 High / Medium / Low로 판정한다.
### High (즉시 확인 필요)
아래 중 하나 이상에 해당하면 High로 판정한다.
- 오늘 마감 또는 24시간 이내 처리가 필요한 메일
- 교수, 팀원, 회사, 공식 기관 등 중요 발신자의 답장 필요 메일
- 사용자 관심사(전공 학사 일정, 관심 금융 티커 등)와 직접 관련된 긴급 메일
- 금융·결제·계정·보안과 관련된 즉시 대응이 필요한 메일
### Medium (오늘 안에 확인 권장)
아래 중 하나 이상에 해당하고 High 조건이 아니면 Medium으로 판정한다.
- 3~7일 이내 마감 예정 메일
- 학교/업무 관련 일반 메일
- 답장이 필요하지만 긴급하지 않은 메일
- 첨부파일 또는 일정 확인이 필요한 메일
### Low (나중에 확인 가능)
아래에 해당하면 Low로 판정한다.
- 광고/프로모션 메일
- 일반 뉴스레터
- 단순 알림 메일
- 이미 처리되었거나 조치가 필요 없는 메일
### 판정 원칙
- 여러 조건에 해당하면 가장 높은 등급을 우선 적용한다.
- 판단에 사용한 신호를 중요도 판단 이유에 반드시 명시한다.
---
## 긴급도
긴급도는 마감·처리 시급성을 기준으로 높음 / 중간 / 낮음으로 표시한다.
