# Payment Analysis 상세 명세 (spec.md)
## 목적
결제·영수증·구독·환불·자동결제 관련 이메일을 분석하여
- 최근 30일 누적 결제 금액 집계
- 구독 서비스 관리
- 결제 예정일 확인
- 비정상 거래 탐지
- 사용자가 확인해야 할 결제 관련 정보를 제공한다.
---
## 입력
### Gmail Connector
- 이메일 제목
- 이메일 본문
- 발신자
- 수신 시간
- 첨부파일 여부

- Mail Summary

## 의존 Skill
### 입력
- Mail Summary

- Security Check

- Morning Mail Brief

## 결제 메일 판별 신호
| 구분 | 신호 예시 |
|---|---|
| 발신자 | 카드사, 은행, 간편결제, 스토어, 구독 서비스 |
| 제목 | 결제, 승인, 영수증, 청구, 자동결제, 구독, 환불 |
| 본문 | receipt, invoice, subscription, payment, renewal |
| 분류 결과 | Mail Classification = 금융 |
| 첨부파일 | Invoice PDF, Receipt PDF |
| 링크 | View Receipt, View Invoice, 결제 상세보기 |

## 추출 항목
| 항목 | 설명 | 없을 경우 |
|---|---|---|
| amount | 결제 금액 | 확인 필요 |
| merchant | 가맹점 또는 서비스명 | 미상 |
| payment_date | 결제일 | 확인 필요 |
| currency | KRW, USD 등 | 명시된 경우만 |
| payment_method | 카드, 계좌 등 | 확인 필요 |
| recurring | 정기결제 여부 | N |
| next_payment_date | 다음 결제 예정일 | 없음 |
---
## 누적 결제 금액 집계 기준
- 최근 30일(1일~30일) 이내 수신된 결제·영수증 메일의 금액을 합산한다.
- 통화가 다른 경우 통화별로 구분하여 집계한다.
- 환불 메일은 해당 금액을 차감 항목으로 표시한다.
- 금액을 확인할 수 없는 메일은 합산에서 제외하고 "확인 필요"로 표시한다.
---
## 비정상 거래 판정 기준
| 상황 | 판정 | 처리 |
|---|---|---|
| 평균 결제액 대비 3배 이상 | 주의 필요 | 사유 표시 |
| 해외 결제 | 주의 필요 | 사용자 확인 권고 |
| 낯선 가맹점 | 주의 필요 | 사용자 확인 권고 |
| 동일 금액 반복 결제 | 주의 필요 | 중복 결제 가능성 표시 |
| Security Risk HIGH 이상 | 경고 | 결제 정보 신뢰 금지 |
| 금액·날짜 확인 불가 | 확인 필요 | 안전 단정 금지 |
---
## 구독 점검 권고 기준
- 오래 사용하지 않은 구독 서비스
- 자동 갱신 예정 서비스
- 동일 서비스 중복 구독 가능성
- 구독 금액이 최근 증가한 경우

## Human-in-the-loop
자동으로 수행하지 않는다.
- 결제 승인
- 결제 취소
- 환불 요청
- 구독 해지
- 카드 정지
- 메일 삭제
모든 행동은 사용자에게 **권고**까지만 제공한다.
