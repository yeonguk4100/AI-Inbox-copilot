# Security Check

## 역할

Gmail 메일의 보안 위험도를 분석하여 피싱, 사칭, 악성 링크 및 위험 첨부파일 여부를 판단한다.

이 Skill은 **보안 분석만 수행**하며 메일 요약, 메일 분류, 중요도 판단, 행동 추천은 수행하지 않는다.

---

## 입력

다음 정보를 입력받는다.

- 메일 제목
- 발신자 이름
- 발신자 이메일
- 수신 시간
- 메일 본문
- 링크(URL)
- 첨부파일 정보

---

## 수행 절차

### 1. 발신자 분석

다음을 확인한다.

- 공식 도메인 여부
- 발신자 이메일 이상 여부
- 발신자 이름과 이메일 불일치 여부
- Reply-To 정보(확인 가능한 경우)

---

### 2. 메일 내용 분석

다음을 확인한다.

- 긴급성을 유도하는 표현
- 개인정보 요구
- 비밀번호 변경 요구
- 로그인 인증 요구
- 결제 또는 송금 요구
- 기관 또는 기업 사칭
- 과도한 혜택 또는 당첨 안내
- 문법적으로 부자연스러운 표현

---

### 3. 링크 분석

다음을 확인한다.

- HTTPS 사용 여부
- 단축 URL 사용 여부
- 공식 도메인 여부
- 의심스러운 URL 패턴

---

### 4. 첨부파일 분석

다음을 확인한다.

- 실행 파일(.exe)
- 스크립트(.js, .vbs)
- 압축파일(.zip, .rar)
- 매크로 포함 문서
- 기타 위험 확장자

---

### 5. 위험도 계산

위험 요소를 종합하여 Security Score를 계산한다.

Security Score는 반드시 **0~100** 범위로 보정한다.

---

### 6. Risk Level 결정

다음 기준을 사용한다.

| Score | Level |
|--------|-------|
| 0~20 | SAFE |
| 21~40 | LOW |
| 41~60 | MEDIUM |
| 61~80 | HIGH |
| 81~100 | CRITICAL |

---

### 7. 피싱 가능성 판단

다음 중 하나를 선택한다.

- 낮음
- 중간
- 높음

판단 근거를 반드시 작성한다.

---

## 사용 규칙

- 반드시 입력된 정보만 사용한다.
- 근거 없는 추측은 하지 않는다.
- 위험 요소는 반드시 설명한다.
- Security Score는 항상 표시한다.
- Risk Level은 반드시 표시한다.
- 불확실한 내용은 **확인 필요**로 표시한다.
- 링크를 직접 실행하거나 첨부파일을 열었다고 가정하지 않는다.
- 메일 삭제, 차단, 격리 등의 자동 조치는 수행하지 않는다.
- Markdown 형식을 유지한다.

## 출력 품질 기준

출력 결과는 다음 조건을 만족해야 한다.

- 위험도 계산이 일관적이다.
- 모든 위험 판단에 근거가 있다.
- 사용자에게 이해하기 쉬운 표현을 사용한다.
- 다음 Skill이 바로 사용할 수 있도록 구조화된 정보를 제공한다.
