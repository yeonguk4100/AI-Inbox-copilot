# 🛡 Security Check Specification

## 목표

Gmail에서 수집한 이메일의 보안 위험도를 분석하여 사용자가 피싱, 스팸, 사칭, 악성 링크 및 위험 첨부파일을 쉽게 식별할 수 있도록 한다.

이 Skill은 메일의 **보안 판단만 수행**하며, 메일 요약, 분류, 중요도 판단 및 행동 추천은 수행하지 않는다.

---

# 입력(Input)

Gmail Connector를 통해 전달받은 메일 정보

- 메일 제목 (Title)
- 발신자 이름 (Sender Name)
- 발신자 이메일 (Sender Email)
- 수신 시간 (Received Time)
- 메일 본문 (Body)
- 포함된 링크(URL)
- 첨부파일 정보 (파일명, 확장자)

---

# 출력(Output)

Markdown 형식의 보안 분석 결과

반드시 아래 항목을 포함한다.

- 메일 정보
- 보안 위험도(Security Score)
- 위험 등급(Risk Level)
- 위험 요소 분석
- 피싱 가능성
- 권장 보안 조치
- 다음 Skill 전달 정보

---

# 성공 기준

- 피싱·사칭·악성 링크·위험 첨부파일을 탐지한다.
- 모든 위험 판단에는 명확한 근거를 포함한다.
- Security Score는 반드시 0~100 범위로 계산한다.
- Risk Level은 기준에 따라 일관되게 분류한다.
- 확신할 수 없는 경우 "확인 필요"를 표시한다.
- 메일 삭제, 격리 등의 자동 조치는 수행하지 않는다.

---

# 작업 절차

1. Gmail Connector에서 메일 정보를 입력받는다.
2. 메일 제목을 분석한다.
3. 발신자 이름 및 이메일 주소를 확인한다.
4. 메일 본문을 분석한다.
5. 포함된 URL을 검사한다.
6. 첨부파일 정보를 확인한다.
7. 아래 위험 요소를 기준으로 분석한다.
8. Security Score를 계산한다.
9. Security Score를 0~100 범위로 보정한다.
10. Risk Level을 결정한다.
11. 피싱 가능성을 판단한다.
12. 권장 보안 조치를 작성한다.
13. 다음 Skill에서 사용할 정보를 생성한다.
14. 출력 형식이 올바른지 검증한다.

---

# 위험 요소 분석 기준

## 발신자

- 공식 도메인 여부
- 발신자 이메일 이상 여부
- Reply-To 정보 불일치 여부(확인 가능한 경우)

---

## 메일 내용

- 긴급성을 유도하는 표현
- 개인정보 요구
- 비밀번호 변경 요구
- 로그인 인증 요구
- 결제 또는 송금 요구
- 과도한 혜택 또는 당첨 안내
- 기관 또는 기업 사칭
- 문법적으로 부자연스러운 표현

---

## 링크

- HTTPS 사용 여부
- 단축 URL 사용 여부
- 공식 도메인 여부
- 의심스러운 URL 패턴

---

## 첨부파일

- 실행 파일(.exe)
- 스크립트(.js, .vbs)
- 압축파일(.zip, .rar)
- 매크로 포함 문서
- 기타 위험 확장자

---

# Security Score 기준

| 조건 | 점수 |
|------|------:|
| 공식 도메인이 아님 | +20 |
| 긴급성 유도 표현 | +15 |
| 개인정보 요구 | +20 |
| 로그인 인증 요구 | +15 |
| 결제/송금 요구 | +15 |
| 의심스러운 링크 | +15 |
| 단축 URL | +10 |
| 위험 첨부파일 | +20 |
| 기관 또는 기업 사칭 | +20 |

최종 점수는 100점을 초과할 수 없다.

---

# Risk Level 기준

| Score | Risk Level |
|--------|------------|
| 0~20 | SAFE |
| 21~40 | LOW |
| 41~60 | MEDIUM |
| 61~80 | HIGH |
| 81~100 | CRITICAL |

---

# 사용 규칙

- Security Score는 반드시 표시한다.
- Risk Level을 반드시 표시한다.
- 위험 요소는 반드시 근거와 함께 작성한다.
- 확신할 수 없는 내용은 "확인 필요"로 표시한다.
- 메일 삭제 또는 격리를 수행하지 않는다.
- 링크를 직접 실행하거나 첨부파일을 열지 않는다.
- Markdown 형식을 유지한다.
